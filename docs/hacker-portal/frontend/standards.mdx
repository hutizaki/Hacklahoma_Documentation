# Frontend Standards

- **Coding standards** — TypeScript, React hooks rules, component structure. See repo: `frontend/STANDARDS.md`, `docs/development/coding-standards.md`.
- **Tailwind** — Utility-first; prefer Tailwind classes; avoid inline styles where possible.
- **Components** — Reusable components in `components/`; common, Admin, Account, etc.

Keep accessibility and responsive behavior in mind for new components.
